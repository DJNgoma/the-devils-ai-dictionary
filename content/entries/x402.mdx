plainDefinition: |
  x402 is an open payment protocol built around HTTP `402 Payment Required`. A server can return payment requirements with a response, after which a client pays through a supported digital-asset rail, supplies proof, and retries the request.
whyExists: |
  APIs and autonomous clients often need accounts, subscriptions, API keys, or human checkout before a small purchase. x402 aims to make payment a native part of the request-response exchange so software can buy access to data, compute, or services as needed.
misuse: |
  It is described as frictionless machine commerce, which carefully omits wallet custody, network fees, exchange risk, sanctions controls, refunds, tax, failed settlement, and the person eventually asked to explain the wallet activity.
practicalMeaning: |
  The service declares an amount, asset, network, recipient, and payment scheme. The client decides whether it may pay, obtains or submits settlement proof, and retries. Production systems still need spending policy, identity, receipts, reconciliation, dispute handling, and idempotent delivery.
