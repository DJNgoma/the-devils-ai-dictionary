  Let the model write the orchestration code, thereby saving round trips and concentrating the incident report.
plainDefinition: |
  Programmatic tool calling lets a model write code that invokes tools inside a controlled execution container. Intermediate tool results can be filtered, combined, or aggregated there before only the useful output is returned to the model.
whyExists: |
  A workflow with many ordinary tool calls requires repeated model turns and can pour large intermediate results into context. Code can perform loops, branching, joins, and filtering more cheaply while preserving the model for planning and interpretation.
misuse: |
  It is described as though fewer model round trips mean less complexity. The complexity has moved into generated code, container policy, tool permissions, runtime limits, and debugging traces—places where it can now fail at machine speed.
practicalMeaning: |
  The model writes a script, a sandbox runs it, permitted tools execute through controlled call paths, and only selected results return to context. Teams need explicit allowlists, time and resource limits, auditable calls, deterministic fixtures, and a firm answer to whether generated code may touch the network.
