  A self-hosted front door for Claude, installed so the organisation may decide who enters and receive one consolidated bill for asking.
plainDefinition: |
  Claude apps gateway is an Anthropic-built, self-hosted control plane for governing Claude Code and Claude Desktop in AWS environments. It centralises identity, routing, model and tool policy, telemetry, and spending controls across Amazon Bedrock and Claude Platform on AWS.
whyExists: |
  Enterprise use of desktop and coding assistants creates fragmented authentication, provider access, policy, observability, and cost management. A gateway gives administrators one managed path while keeping the control plane inside their AWS account.
misuse: |
  The gateway is treated as a model, a universal Claude enterprise tier, or proof that data and tools are safe. Central routing improves control; it does not repair excessive permissions, weak policy, retained prompts, or an unsafe downstream provider configuration.
practicalMeaning: |
  Organisations deploy a stateless gateway container backed by PostgreSQL, connect corporate identity, define model and tool rules, and point supported Claude apps through it. They must still operate the service, secure its credentials and database, inspect provider terms, and test failure behaviour.
