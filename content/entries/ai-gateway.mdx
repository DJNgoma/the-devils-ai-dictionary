  The toll booth installed after everyone realised "just call the model" was not a governance strategy.
plainDefinition: |
  An AI gateway is an intermediary service that routes AI requests between applications and model providers, usually adding authentication, logging, rate limits, retries, caching, policy controls, and cost tracking.
whyExists: |
  Teams need a common control point for many applications, models, providers, and budgets. A gateway makes it possible to change providers, enforce policy, observe usage, and prevent every product team from inventing its own fragile little tunnel.
misuse: |
  It gets marketed as an instant platform. A gateway can centralise traffic, but it cannot decide product quality, fix bad prompts, or turn unclear governance into architecture by standing in the middle.
practicalMeaning: |
  In practice it means one place to manage API keys, provider routing, request metadata, redaction, quotas, fallbacks, and dashboards. It also means one more critical dependency that must be boringly reliable.
example: |
  "The AI gateway earned its keep when the provider outage became a routing event instead of a company-wide treasure hunt for hard-coded API keys."
