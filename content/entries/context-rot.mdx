devilDefinition: |
  The slow corruption of a long AI session as every clarification, exception, and half-fixed mistake is preserved with the reverence of scripture.
plainDefinition: |
  Context rot is the degradation that happens in long-running chats or agent workflows when accumulated instructions, stale assumptions, noisy history, and partial fixes start reducing output quality.
whyExists: |
  People needed a name for the very practical observation that more context is not always better context. Once a session becomes long enough, the model may carry forward the wrong facts with admirable loyalty.
misuse: |
  The phrase gets used to blame the model for every bad answer in a long thread, including cases where the human supplied contradictory instructions, never reset the task, or kept shovelling debris into the prompt like a municipal service.
practicalMeaning: |
  In practice it usually means the session needs pruning, a fresh summary, or a clean restart. For agent systems it often means state management is doing a worse job than the demo admitted.

  "By the eighth refinement the coding assistant had context rot: it remembered every detour and none of the point."

Context rot is what happens when people confuse retention with understanding. A long memory is only helpful if someone occasionally removes the rubbish.
