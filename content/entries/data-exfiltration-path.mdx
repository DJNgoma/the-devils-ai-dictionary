devilDefinition: |
  The route sensitive information takes when a helpful system forgets that not every destination is a colleague.
plainDefinition: |
  A data exfiltration path is a route by which sensitive data can leave its intended boundary through prompts, logs, tools, connectors, retrieved content, generated output, or external services.
whyExists: |
  AI systems connect models to data and tools. Every new connector, context source, logging stream, browser action, and summary endpoint can become a path for information to move where it should not.
misuse: |
  It gets reduced to dramatic breaches, while quieter paths hide in analytics logs, prompt traces, vendor calls, screenshots, exported summaries, and tools that were granted too much scope.
practicalMeaning: |
  In practice it means mapping data flows, limiting tool permissions, redacting logs, isolating untrusted content, and treating generated summaries as possible carriers of sensitive information.
example: |
  "The data exfiltration path was not the model remembering a secret; it was the agent dutifully pasting the secret into a third-party ticket it had permission to create."
