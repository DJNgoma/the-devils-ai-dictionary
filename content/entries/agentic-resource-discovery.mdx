  A directory for agents that have finally outgrown discovering tools by reading the integration team's bookmarks.
plainDefinition: |
  Agentic Resource Discovery is an open specification for publishing, discovering, and verifying tools, APIs, skills, services, and agent endpoints across organisational boundaries. It adds a discovery layer before execution protocols such as MCP, OpenAPI, or A2A are used.
whyExists: |
  Agent capabilities are scattered across providers, registries, and private systems. ARD lets domains publish machine-readable catalogues that discovery services can index, so clients can locate resources by task rather than relying only on hard-coded integrations.
misuse: |
  Discovery is confused with permission or trust. Finding a signed catalogue and confirming domain ownership does not decide whether a tool is safe, authorised for this user, suitable for the task, or honest about what it does.
practicalMeaning: |
  Providers publish an `ai-catalog.json` manifest; registries index those catalogues; and clients search for capabilities before connecting through an execution protocol. Operators still need authentication, authorisation, ranking policy, provenance checks, monitoring, and revocation.
