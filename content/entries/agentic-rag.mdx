  RAG with an errand runner: it can search again, think about the bad search, and occasionally make the mess more methodical.
plainDefinition: |
  Agentic RAG is a retrieval-augmented generation pattern where an agent plans, searches, filters, retries, compares sources, or calls retrieval tools iteratively before producing an answer.
whyExists: |
  Some questions are too complex for a single search. They need decomposition, multiple queries, source comparison, follow-up retrieval, and a decision about when the evidence is good enough to stop.
misuse: |
  It is used as a prestige label for any RAG system with more than one step. Extra loops do not automatically improve grounding. They can add latency, cost, failure paths, and confidence in a more elaborate mistake.
practicalMeaning: |
  In practice it means query planning, retrieval tools, loop limits, reranking, source triangulation, provenance capture, fallback behaviour, and evaluations that prove the extra agency is earning its keep.
example: |
  "The agentic RAG system answered the policy question by searching the handbook, checking the latest memo, rejecting an obsolete clause, and still needing an eval to prove this was not just expensive browsing."
