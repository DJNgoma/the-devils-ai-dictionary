  The tasteful removal of yesterday's tool output before today's agent mistakes it for scripture.
plainDefinition: |
  Context editing selectively removes or replaces content in a growing conversation history before the next model call. Current implementations can clear old tool results or thinking blocks while preserving more recent information needed for continuity.
whyExists: |
  Long-running agents accumulate file contents, search results, tool traces, and reasoning state until cost rises or the context limit approaches. Curating stale material can preserve useful working space without restarting the entire session.
misuse: |
  It is sold as memory management while sometimes functioning as deletion with reassuring typography. Clearing the wrong evidence can break continuity, hide an earlier constraint, invalidate prompt caches, or leave the model confidently reasoning from the placeholder.
practicalMeaning: |
  Define thresholds and retention rules, clear older low-value blocks, and keep enough recent state for the task. Measure task quality, token savings, cache invalidation, and whether audit logs retain what the model no longer sees.
