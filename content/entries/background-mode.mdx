devilDefinition: |
  An API request allowed to outlive the user's attention span and, with proper engineering, the process that started it.
plainDefinition: |
  Background mode is an asynchronous API execution option for long-running model work. The client starts a response, receives an identifier, and retrieves or receives the result later rather than keeping one request connection open until completion.
whyExists: |
  Agentic and reasoning workloads can run longer than ordinary HTTP timeouts or interactive user sessions. Background execution gives them a managed lifecycle with status, cancellation, polling, and eventual result retrieval.
misuse: |
  It is described as though putting work in the background makes it durable, observable, or safe. Those properties depend on the provider contract and the application's handling of retries, cancellation, duplicate work, retention, and partial failure.
practicalMeaning: |
  The product must store the response identifier, expose progress honestly, recover after restarts, decide how often to poll, handle expiry and failure, and tell the user when the result is ready. It is asynchronous execution, not an exemption from lifecycle design.
