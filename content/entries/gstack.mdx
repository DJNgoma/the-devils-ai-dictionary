devilDefinition: |
  The proposition that one founder, one agent shell, and enough specialised slash commands can now impersonate a small product org with alarming speed.
plainDefinition: |
  gstack is Garry Tan's open-source workflow and skill stack for Claude Code and related agent hosts, packaged as a large set of opinionated roles and commands for planning, coding, review, QA, security, release, and retrospectives.
whyExists: |
  Because agent users do not just want a model. They want a repeatable operating system for product work: who reviews, who tests, who ships, and what sequence keeps the chaos looking intentional.
misuse: |
  By talking as if the stack itself is a substitute for product judgement, engineering taste, or organisational memory. A thick layer of specialist prompts can accelerate work. It cannot abolish the need to know what good looks like or who is accountable when it is not.
practicalMeaning: |
  In practice it means turning an agent shell into a highly structured software-factory workflow, with preset roles for founder review, engineering review, design critique, browser QA, security audit, release management, and post-mortems.

gstack is notable because it stops pretending the agent is just "helping with code". It formalises the wider ambition: compressing a startup's rituals into reusable command surfaces and seeing how much of the organisation can be simulated before management notices.
