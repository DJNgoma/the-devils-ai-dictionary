  Procedural memory in agent systems is retained knowledge of how to perform tasks—skills, playbooks, tool sequences, and habits—as distinct from episodic records of what happened in a particular run.

  Agents that only store transcripts relearn the same steps. Teams wanted a name for reusable how-to knowledge that can be loaded as skills or policies.

  Treated as guaranteed competence. A stored procedure can be outdated, unsafe, or optimised for the wrong environment.

  Version skills, scope when they apply, test them like code, and separate them from raw conversation logs. Prefer explicit skill libraries over mystery memory blobs.
