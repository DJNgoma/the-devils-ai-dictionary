  A way for shopping agents to show their papers at the merchant's door, issued by a payment network with an understandable interest in the shopping.
plainDefinition: |
  Trusted Agent Protocol is a Visa-led framework for letting commerce agents present cryptographically verifiable identity and transaction context to merchants. It is intended to help merchants distinguish recognised agent traffic from impersonation or unauthorised automation.
whyExists: |
  Web shops traditionally infer trust from browser sessions, user actions, accounts, and fraud signals. Agents disturb those assumptions by browsing and buying on someone's behalf, so merchants need stronger evidence about the agent, its operator, and the authority attached to a request.
misuse: |
  The word "trusted" is easily promoted from authenticated to benevolent. A valid agent identity does not prove that the user approved this purchase, that the recommendation was sound, or that the merchant should permit every requested action.
practicalMeaning: |
  Participating agents attach signed credentials or request evidence that merchants can verify and map to policy. Deployment still requires user consent, authorisation limits, replay protection, key rotation, fraud controls, audit trails, and a decision about which issuers the merchant trusts.
example: |
  "TAP confirmed that the visitor was a recognised shopping agent, leaving the merchant to decide whether it should be allowed near the refund policy."
