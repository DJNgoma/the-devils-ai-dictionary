devilDefinition: |
  A way of making tools and resources speak a common dialect so the assistant stops needing a bespoke adapter for every cupboard in the house.
plainDefinition: |
  MCP stands for Model Context Protocol. It is a protocol for exposing tools, resources, and related capabilities to AI clients in a standardised way, so assistants can discover and use external systems without each integration being entirely custom.
whyExists: |
  AI applications increasingly need access to files, APIs, documentation, databases, prompts, and tools. Without a shared protocol, every client-server pairing becomes its own integration project. MCP exists to reduce that fragmentation.
misuse: |
  It gets spoken about as though a protocol itself makes systems intelligent, secure, or interoperable by magic. It does not. Standardisation helps, but permissions, trust boundaries, authentication, and product design still matter more than the acronym.
practicalMeaning: |
  In practice it usually means exposing capabilities through a consistent interface so a client can list resources, call tools, and work with external context more systematically. The real value is reducing bespoke glue and making integrations more legible.
example: |
  "MCP did not solve the assistant's judgement problem. It solved the quieter and more useful problem of not writing one-off connectors forever."

Protocols are boring in the most productive possible sense. They do not remove the need for judgement. They remove some of the repetitive integration labour that distracts from the judgement problem.
