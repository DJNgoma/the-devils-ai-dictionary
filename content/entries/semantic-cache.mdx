devilDefinition: |
  A way of avoiding another model call by deciding this new question is close enough to the last one to deserve the same invoice-saving answer.
plainDefinition: |
  A semantic cache stores earlier model responses and reuses them when a new request is sufficiently similar in meaning, usually based on embeddings or another similarity measure, instead of sending the request back to the model.
whyExists: |
  Model calls cost money and time. Semantic caches exist because many user questions are near-repeats, and it is wasteful to buy fresh intelligence every time someone asks the same thing in a slightly different shirt.
misuse: |
  It gets presented as a free latency win, as though similarity were the same thing as correctness. A cache can be fast and still be wrong, stale, or impolitely blind to user-specific context.
practicalMeaning: |
  In practice it means choosing an embedding model, setting similarity thresholds, deciding what context is part of the cache key, expiring results before they fossilise, and accepting that a cheaper answer can still be the wrong one.

Semantic caches are useful precisely because they are not magical. They are ordinary infrastructure masquerading as cleverness, which is often the most reliable kind.
