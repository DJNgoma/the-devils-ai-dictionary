devilDefinition: |
  The temporary structure teams build around a model so the workflow can stand upright before the surrounding software has fully learned to do its job.
plainDefinition: |
  A prompt scaffold is the structured set of instructions, examples, delimiters, and output constraints used to support a workflow without being the workflow itself.
whyExists: |
  Models respond better when tasks are framed clearly. Prompt scaffolds exist because careful structure can stabilise behaviour, reduce ambiguity, and make an immature workflow usable before every surrounding control has been turned into software.
misuse: |
  It gets treated as permanent architecture. A scaffold is helpful precisely because it is a support structure, not a substitute for real validation, permissions, retrieval quality, or product logic. When a team starts asking the prompt to carry the whole building, the prompt begins to sag.
practicalMeaning: |
  In practice it usually means ordered instructions, explicit roles, few-shot examples, output schemas, reminders about tool use, and the gradual discovery that some recurring prompt discipline deserves to migrate into code, policy, or interface design.
example: |
  "The prompt scaffold made the assistant usable. The trouble began when the team mistook that usability for a reason never to build the missing product controls."

Prompt scaffolds are respectable. The pathology begins when they become load-bearing for problems the application ought to own more directly.
