plainDefinition: |
  Agentic retrieval is a retrieval pipeline that uses a model to plan or decompose a complex question into focused subqueries, executes those searches, reranks and merges the results, and returns grounding data or an answer with source references.
whyExists: |
  A single search query often misses implied context, multiple intents, or evidence spread across sources. Planning several targeted queries can improve coverage and give downstream agents stronger grounding for complex questions.
misuse: |
  The adjective "agentic" can conceal an expensive query fan-out behind a fashionable noun. More searches do not guarantee better evidence; they can multiply latency, irrelevant chunks, ranking errors, and token charges with admirable initiative.
practicalMeaning: |
  Feed the user question and conversation history to a planner, run the resulting keyword, vector, or hybrid subqueries, rerank them, merge the best evidence, and preserve an activity log. Evaluate answer quality against classic retrieval as well as latency, cost, source diversity, and failure on deceptively simple questions.
