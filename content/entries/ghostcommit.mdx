  A malicious instruction hidden in an image for the code-review agent that helpfully reads what the human reviewer cannot see.
plainDefinition: |
  Ghostcommit is a named multimodal prompt-injection attack against AI-assisted software development and code review. An attacker places instructions in repository images that appear benign to people but can steer a vision-capable agent towards unsafe tool use, secret access, or supply-chain compromise.
whyExists: |
  Modern coding agents inspect more than source text: they may analyse screenshots, diagrams, documentation assets, issues, and pull-request media while holding repository or CI permissions. Those extra input channels create an instruction surface that ordinary code review may overlook.
misuse: |
  Ghostcommit is treated as a peculiar image bug in one product. The broader problem is untrusted multimodal content crossing into an agent's instruction context while the agent has consequential tools; filtering one encoding does not repair the trust boundary.
practicalMeaning: |
  Treat every repository artefact as untrusted data, isolate multimodal analysis from privileged execution, minimise credentials, require approval for sensitive actions, monitor outbound access, and test code-review agents with adversarial images as well as adversarial text.
example: |
