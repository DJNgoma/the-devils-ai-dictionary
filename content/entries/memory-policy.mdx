  The quiet constitution deciding what the system remembers, for how long, and which future mistake inherits the privilege.
plainDefinition: |
  Memory policy is the set of rules governing what an AI system stores from interactions, how long it keeps that information, when it retrieves it, and when it must forget or redact it.
whyExists: |
  It exists because continuity is useful and indiscriminate memory is dangerous. Long-running systems can become more helpful by remembering preferences, facts, and prior decisions, but they can also become more wrong, more invasive, and more difficult to debug.
misuse: |
  The term gets reduced to "we have memory now", as if any retained context were inherently intelligent. It also gets used to dress up ordinary logging or vector storage without answering the harder questions about consent, correction, deletion, and sensitive data.
practicalMeaning: |
  In practice it means retention windows, summarisation rules, provenance, retrieval thresholds, redaction filters, user controls, and clear logic for when stale memory should be ignored rather than honoured out of misplaced loyalty.

  "The assistant felt personal right up until its memory policy allowed one outdated preference to haunt every recommendation for the next three months."

Memory policy matters because forgetting is part of quality control. A system that never forgets is often just a system that never learned restraint.
