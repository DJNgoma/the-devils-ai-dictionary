plainDefinition: |
  MCP sampling is the protocol capability that lets an MCP server request model generation through the client, rather than directly calling a model itself.
whyExists: |
  Some servers need model assistance during a workflow but should not own model credentials, user context, or policy decisions. Sampling lets the client mediate that generation request.
misuse: |
  It is easy to describe as the server "using the model", which hides the important part: the client remains the place where user consent, model choice, policy, and context should be controlled.
practicalMeaning: |
  In practice it means the client must decide whether a server's generation request is allowed, what context may be included, and how the resulting text is shown, logged, or passed back.
