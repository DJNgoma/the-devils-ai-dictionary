  Agent credential sprawl is the uncontrolled growth of tokens, keys, sessions, service accounts, delegated grants, and other credentials created so agents can act across tools and systems.
whyExists: |
  Agents become useful by connecting to systems that require authority. Each connector, workspace, user, environment, vendor, and workflow can produce another credential. Without lifecycle management, the access estate grows faster than the team's memory of why it exists.
misuse: |
  It is dismissed as ordinary secret management. Agent credentials are often bound to dynamic goals, delegated user authority, tool calls, and automated side effects, so stale or overbroad access can do more than merely sit there looking untidy.
practicalMeaning: |
  In practice it means inventorying agent tokens, mapping them to owners and workflows, narrowing scopes, rotating secrets, expiring delegated grants, separating environments, and revoking the credentials that survived because nobody wanted to break the demo.
