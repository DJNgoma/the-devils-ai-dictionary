  Prompt injection is a failure mode or attack in which untrusted text changes a model's behaviour by smuggling in instructions that override, confuse, or redirect the intended prompt.
whyExists: |
  Because language models read instructions, data, and hostile text through the same basic channel. They do not naturally maintain the clean boundaries software engineers would very much prefer.

  By treating it as an exotic hacker trick rather than a structural property of model-based systems, or by using it to describe every wrong answer regardless of whether instructions were actually manipulated.

  In practice it means any tool-using or retrieval-heavy system must assume that documents, web pages, emails, and user content may contain adversarial instructions that need filtering, isolation, or refusal.

Prompt injection matters because it collapses a comfortable fiction. The model does not reliably know which text is policy, which is data, and which is a trap wearing the manners of documentation.
