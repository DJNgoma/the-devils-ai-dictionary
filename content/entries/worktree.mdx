plainDefinition: |
  A worktree is a linked working tree attached to the same Git repository, allowing different branches or tasks to be checked out in separate directories at the same time.
whyExists: |
  It exists because parallel work is useful and mixing every branch, experiment, dependency state, and agent task into one checkout is how small mistakes become local archaeology.
misuse: |
  It gets treated as a sandbox. It is not one. A worktree isolates files and branches, not credentials, networks, databases, or the agent's ability to run a regrettable command.
practicalMeaning: |
  In AI-assisted development, it usually means giving a coding agent or subagent a separate branch and directory so it can edit, test, and report back without colliding with the main checkout or another agent's work.

  "The worktree let three agents work in parallel, which was helpful until everyone remembered they still shared one confused package cache."

Worktrees are boring in the way useful tools often are. They do not make agents safe. They make their edits easier to contain, inspect, and throw away.
