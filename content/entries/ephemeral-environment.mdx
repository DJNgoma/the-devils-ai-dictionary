  A temporary machine for agent code that dies after the task, taking its mess—and hopefully its secrets—with it.
plainDefinition: |
  An ephemeral environment is a short-lived runtime (container, VM, or worktree sandbox) created for an agent or job to execute untrusted or experimental work, then destroyed.
whyExists: |
  Agents write and run code. Durable shared machines accumulate risk; ephemeral ones limit persistence of compromise and clutter.
misuse: |
  Called ephemeral while mounting production credentials and long-lived volumes. Destruction without secret hygiene is theatre.
practicalMeaning: |
  Provision minimal identity, no standing prod access, network policy, and automatic teardown. Pass only needed artefacts out.
example: |
  "The ephemeral environment compiled the patch and vanished before anyone could SSH in to 'just check.'"
