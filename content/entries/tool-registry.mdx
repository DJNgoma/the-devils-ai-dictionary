  A menu of powers offered to an agent, ideally written by someone who understands the difference between capability and liability.
plainDefinition: |
  A tool registry is a structured catalogue of tools an AI system can discover or use, including each tool's name, schema, permissions, owner, description, and operational constraints.
whyExists: |
  Agents need a reliable way to know which actions are available and how to call them. As tool ecosystems grow, teams need registries to avoid stale schemas, duplicated integrations, hidden permissions, and the old tradition of guessing from a README last touched during a launch week.
misuse: |
  It gets treated as a plugin marketplace with fewer consequences. Registering a tool does not make it safe, maintained, authorised, or suitable for an agent that has just been persuaded by a malicious PDF.
practicalMeaning: |
  In practice it means schema versioning, permission boundaries, ownership, environment separation, audit logs, deprecation rules, and a process for deciding which tools an agent sees in a given task.

  "The tool registry made the agent less adventurous by removing three obsolete endpoints and one payroll action nobody remembered approving."
