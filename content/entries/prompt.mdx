devilDefinition: |
  A politely worded attempt to steer a probability machine into behaving as if it has read the brief and cares.
plainDefinition: |
  A prompt is the input text, and sometimes structured context, provided to a model in order to shape its output. In LLM systems this can include instructions, examples, retrieved documents, tool results, formatting rules, and the user's actual request.
whyExists: |
  The term exists because the way you frame input materially affects output. Language models do not simply receive a question. They respond to the entire text context they are given.
misuse: |
  It gets treated either as wizardry or as trivia. Both are wrong. Prompt quality matters, but prompting is not a substitute for missing product design, missing data, or missing control logic. A good prompt cannot rescue a bad system forever.
practicalMeaning: |
  In practice it usually means the assembled context sent to the model: system instructions, user text, retrieved snippets, examples, tool schemas, output rules, and maybe the remnants of last week's prompt experiments still haunting production.
example: |
  "The issue was not that the model was weak; the prompt was half policy manual, half legal disclaimer, and no longer resembled a usable instruction."
