  The awkward discovery that a model cannot leak a document it was never allowed to fetch, which is precisely why everyone should check.
plainDefinition: |
  Retrieval permissions are the access-control rules that decide which documents, records, chunks, or sources a retrieval system may return for a particular user, agent, tenant, or task.
whyExists: |
  RAG systems can search across private corpora faster than organisations can explain their folder permissions. If the retriever ignores access control, the model may faithfully summarise information the requester should never have seen.
misuse: |
  Teams apply security after generation, as if a prompt instruction could redact whatever the retrieval layer already exposed. Another common mistake is building one shared index and hoping group memberships remain somebody else's problem.
practicalMeaning: |
  In practice it means synchronising document ACLs, enforcing tenant and group filters at retrieval time, handling revocation, testing adversarial queries, auditing blocked sources, and making permission checks part of the retrieval path rather than a footnote.

  "The HR chatbot improved dramatically once retrieval permissions stopped it from treating executive compensation folders as helpful context for everyone."
