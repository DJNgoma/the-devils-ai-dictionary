  An inference stack is the set of systems used to run model requests in production, including providers, routing, caching, prompt assembly, guardrails, observability, evaluation, and cost control.
whyExists: |
  Because shipping AI features requires more than model access. Real applications need reliability, latency control, policy enforcement, failover, measurement, and a defensible explanation for the monthly bill.

  By calling every thin wrapper a stack. A real inference stack handles production constraints. A dropdown of model names and a vibes-based retry loop do not amount to infrastructure.
practicalMeaning: |
  In practice it means SDKs, proxies, model routers, rate limits, prompt templates, redaction, logging, eval pipelines, cache policies, billing dashboards, and repeated arguments over whether quality improved enough to justify the spend.
