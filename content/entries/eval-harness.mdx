devilDefinition: |
  The machinery teams build once they realise strong opinions about model quality are not, in fact, a release process.
plainDefinition: |
  An eval harness is the framework or runner used to execute repeatable evaluations against models, prompts, tools, or agent workflows and collect comparable results.
whyExists: |
  It exists because manual spot checks do not scale, model behaviour drifts, prompts regress quietly, and agent systems fail in more ways than a demo can comfortably admit.
misuse: |
  The harness gets mistaken for truth when it only reflects the cases you bothered to encode. A tidy dashboard is still compatible with spectacular blind spots.
practicalMeaning: |
  In practice it usually means versioned datasets, scorers, task traces, cost accounting, regression thresholds, and a growing pile of awkward examples nobody wants to drop because they keep catching something embarrassing.
example: |
  "The eval harness did not make the agent good. It merely prevented the team from declaring victory after each plausible-looking run."
