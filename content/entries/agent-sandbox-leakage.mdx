  Agent sandbox leakage is the unintended movement of data, credentials, files, network access, tool authority, or side effects across the boundary of an environment meant to isolate an agent's work.
whyExists: |
  Useful agents need somewhere to read, write, run code, browse, call tools, and test ideas without being handed full production access. Sandboxes provide that containment, but real workflows often share caches, secrets, mounts, logs, sessions, networks, or approval paths.

  In practice it means checking the unpleasant edges: copied environment files, inherited credentials, shared volumes, outbound network rules, persistent browser profiles, unredacted logs, tool scopes, and any command that turns a contained mistake into an external action.
