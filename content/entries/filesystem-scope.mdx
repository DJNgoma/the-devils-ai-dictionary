  Filesystem scope is the policy defining which directories and files an agent may read or write during a task, often enforced via sandboxes, worktrees, or OS permissions.

  File access is where agents cause durable damage. Scope turns ambient computer access into a defined workspace.

  Set to the user's home directory for convenience. Convenience is how secrets and unrelated repos get rewritten.
practicalMeaning: |
  Default to a task worktree, deny dotfile and secret paths unless required, and log file mutations. Prefer ephemeral copies for experiments.
