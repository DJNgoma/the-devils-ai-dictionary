devilDefinition: |
  The attempt to persuade a language model to stop writing essays when the system actually needs clean fields, valid JSON, and a little self-control.
plainDefinition: |
  Structured outputs are model responses constrained to a predefined schema or format, such as JSON objects with required fields and typed values, so downstream systems can parse and use them more reliably.
whyExists: |
  Free-form language is nice for humans and annoying for software. Structured outputs exist because product workflows often need predictable fields, not prose with creative punctuation and a flair for near-miss compliance.
misuse: |
  They get described as if a schema magically guarantees truth. It does not. Structured nonsense is still nonsense. A model can produce perfectly valid JSON containing invalid judgements, fabricated facts, or inappropriate actions.
practicalMeaning: |
  In practice it usually means schema definitions, validation layers, retries, and careful prompt design so the model returns machine-usable data. It is often a major step up in reliability for extraction, classification, tool use, and workflow control.
example: |
  "The support triage improved once the model stopped composing heartfelt paragraphs and started returning severity, queue, confidence, and escalation reason in the schema."
