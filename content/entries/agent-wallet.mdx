  An agent wallet is a payment or value-control mechanism that lets an AI agent hold, request, or use spending authority within defined limits.
whyExists: |
  Agents that can book travel, buy supplies, trade compute, or pay other services need something more precise than "use the user's card". A wallet can separate authority, budget, identity, approvals, and audit history from the human's everyday account.
misuse: |
  It gets treated as if putting money behind an agent is merely a UX improvement. In reality it changes the risk model: mistaken purchases, fraud, policy bypass, vendor steering, refund handling, and blame assignment all become part of the product.
practicalMeaning: |
  In practice it usually means prepaid balances, spending caps, merchant allowlists, approval thresholds, revocation controls, transaction logs, and a painfully explicit answer to who pays when the agent has been cleverly wrong.
