  A carefully curated scrapbook of past sessions, sold as a mind so the product can claim it remembers you rather than your last JSON blob.
plainDefinition: |
  Episodic memory is a memory design that stores discrete interaction episodes — events, sessions, or task traces — so an agent can retrieve relevant past occurrences rather than replaying an entire chat log as permanent identity.
whyExists: |
  Long-running agents need continuity without drowning in raw transcript. Episodes give structure: what happened, when, with whom, and what outcome mattered enough to keep.
misuse: |
  Every chat message is treated as an episode, or episode stores are marketed as consciousness. Without retention policy, provenance, and user control, "memory" becomes an unbounded personal dossier with a friendly UI.
practicalMeaning: |
  In practice it means segmenting work into episodes, summarising or indexing them, retrieving by relevance rather than recency alone, and letting users inspect, edit, or delete what the system claims happened.
example: |
  "The agent's episodic memory recalled the vendor dispute from March and, less helpfully, the joke the user made about resigning."

Episodic memory is useful when the system remembers the right past and forgets the rest on purpose. Continuity without policy is just surveillance with better nouns.
