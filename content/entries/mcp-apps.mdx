  The moment a tool result decides it deserves controls, state, and a small front-end team.
plainDefinition: |
  MCP Apps is an official Model Context Protocol extension for interactive user interfaces rendered inside an MCP host. A tool declares a UI resource; after the tool call, the host fetches that resource and renders its HTML in a sandboxed iframe within the conversation.
whyExists: |
  Some tool results are awkward as prose or static structured data. Maps, forms, charts, previews, and multi-step controls are easier to understand and operate as interfaces that remain connected to the tool and conversation context.
misuse: |
  It gets described as though returning HTML makes an MCP server a complete application platform. Hosts still decide support, sandboxes constrain behaviour, and the server still owns the unglamorous work of data, authentication, state, and safe tool design.
practicalMeaning: |
  The server exposes a tool plus a referenced UI resource. A compatible host calls the tool, fetches the resource, and renders it under host-controlled security restrictions. Teams must test host compatibility, content security, accessibility, and what happens when the interface is unavailable.
