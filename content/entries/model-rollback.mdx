  The dignified way of saying that production reality declined the invitation to share your release optimism.
plainDefinition: |
  Model rollback is the controlled reversion from a newly deployed model, prompt stack, or routing configuration back to a previous version after regressions, cost surprises, or safety issues appear.
whyExists: |
  AI releases do not fail only by crashing. They also fail by becoming slower, stranger, costlier, or less trustworthy in ways that benchmarks and demos did not predict. The term exists because recovery needs to be designed before the awkwardness arrives.
misuse: |
  It gets imagined as a simple button. Real rollback is harder when model changes came bundled with prompt edits, schema changes, routing tweaks, tool updates, or silently shifted provider behaviour. Reverting the label is not the same as restoring the old system.
practicalMeaning: |
  In practice it usually means version pinning, release gates, shadow tests, archived prompts, configuration history, and evidence good enough to decide whether the problem lives in the model, the workflow, or the surrounding environment.

  "The rollback was delayed because nobody had preserved the previous prompt bundle, proving once again that release confidence and release hygiene are not twins."
