plainDefinition: |
  MCP roots are client-provided locations, such as folders or workspace boundaries, that tell an MCP server which areas are relevant to the current context.
whyExists: |
  Servers often need to understand the project, workspace, or file area a user is working in. Roots provide that boundary without requiring the server to infer relevance from the entire machine or account.
misuse: |
  They get confused with permissions. A root can describe relevant scope, but it does not by itself settle what the server may read, write, index, or send elsewhere.
practicalMeaning: |
  In practice it means tooling can be more context-aware while still needing explicit access controls, user consent, audit logs, and conservative defaults around sensitive files.
example: |
  "The client shared the repository root with the MCP server, which was context, not a romantic commitment to every secret in the home directory."
