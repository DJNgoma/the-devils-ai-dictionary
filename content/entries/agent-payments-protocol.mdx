devilDefinition: |
  A protocol for giving an agent proof of permission to spend, which is more reassuring than a prompt and less reassuring than the first audit.
plainDefinition: |
  Agent Payments Protocol, commonly called AP2, is an open protocol led by Google for agent-initiated payments. It uses cryptographically signed mandates to record what a user authorised, what an agent requested, and the transaction details presented to the payment system.
whyExists: |
  Existing payment flows generally assume a person is present at checkout. Agentic transactions need portable evidence of user intent, bounded delegation, and transaction context so merchants, payment providers, and agents can distinguish an authorised purchase from software improvisation.
misuse: |
  Mandates are sometimes treated as a complete trust framework. They provide evidence about declared authority, but the surrounding system must still protect keys, interpret limits, prevent replay, verify merchant claims, handle fraud, and resolve disputes.
practicalMeaning: |
  A user signs an intent mandate describing permitted action; the agent later produces a cart or transaction mandate with the concrete purchase. Participants verify the signatures and constraints before payment. Implementers still need identity, consent UX, revocation, limits, receipts, and liability rules.
example: |
  "AP2 proved the agent had permission to buy the ticket, though not that it had chosen the sensible airport."
