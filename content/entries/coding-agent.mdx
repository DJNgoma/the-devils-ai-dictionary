  The software assistant that turned "write the code" into the easy part and "own the consequences" into a fresh management discipline.
plainDefinition: |
  A coding agent is an AI system designed to inspect repositories, edit files, run commands, use documentation or browsers, and carry out multi-step software tasks with some degree of autonomy.
whyExists: |
  Autocomplete was helpful, but it still left humans doing the orchestration: finding the right files, running the tests, checking the docs, fixing the follow-on errors, and keeping the task in memory. Coding agents exist because developers wanted delegation, not merely more enthusiastic tab completion.
misuse: |
  It gets used as proof that software engineering has become a prompt-shaped clerical job. In reality the category is only impressive when the surrounding repo is legible, the tests say something useful, and an actual engineer is still responsible for the final judgement.
practicalMeaning: |
  In practice it usually means bounded delegation: background code changes, refactors, bug fixes, test runs, and repo chores completed faster than a person would do them by hand, followed by the entirely traditional need for review, context, and rollback plans.
