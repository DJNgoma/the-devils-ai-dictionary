devilDefinition: |
  Writing software by maintaining morale with the model until something compiles and nobody asks too many follow-up questions.
plainDefinition: |
  Vibe coding is a loose term for building software through conversational prompting, rapid iteration, and acceptance-by-feel rather than explicit design and implementation from first principles.
whyExists: |
  Because code generation became good enough that people can now reach working prototypes, scripts, and interface drafts through steering and selection rather than typing every line themselves.
misuse: |
  By presenting it as a replacement for engineering discipline, or by using it as a sneer for any AI-assisted development at all. The useful version is fast exploratory work. The reckless version is calling the prototype done.
practicalMeaning: |
  In practice it means quick spikes, glue code, scaffolded UI, and a heavy dependence on taste, review, and later cleanup once the generated momentum collides with reality.
