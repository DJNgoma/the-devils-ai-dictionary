plainDefinition: |
  An MCP prompt is a reusable prompt template or prompt-like workflow exposed by an MCP server so a client can discover and invoke it with appropriate arguments.
whyExists: |
  Some integrations need repeatable ways to start tasks, collect inputs, or frame a domain-specific workflow. Exposing prompts through MCP lets servers offer those patterns without hiding them in client-specific code.
misuse: |
  It gets mistaken for business knowledge. A reusable prompt may encode a helpful workflow, but it can also encode yesterday's assumptions with better packaging.
practicalMeaning: |
  In practice it means prompt templates become part of the integration surface and need versioning, ownership, review, and testing like any other operational artifact.
