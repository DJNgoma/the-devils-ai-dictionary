devilDefinition: |
  The discovery that if you feed the machine the right evidence, tools, and constraints, it can briefly look like it knows what it is doing.
plainDefinition: |
  Context engineering is the practice of deciding what instructions, examples, retrieved facts, memory, tools, and state a model receives at run time, and in what form.
whyExists: |
  Because a prompt on its own is usually too thin for serious work. Useful systems need the right surrounding material, not just a clever first sentence.
misuse: |
  By using it as a grander title for prompt writing, or as permission to dump half the company wiki into the context window and pray for synthesis. Good context engineering is selective. Bad context engineering is hoarding with tokens.
practicalMeaning: |
  In practice it means building the retrieval, ranking, compression, state, and tool-selection logic that gives a model enough situational awareness to be useful without drowning it in irrelevant prose.
example: |
  "The demo looked intelligent right up to the point the context engineering failed and the system answered yesterday's problem with last quarter's policy."
