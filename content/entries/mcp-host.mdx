  An MCP host is the application (IDE, desktop agent, chat product) that embeds MCP client capabilities, manages connections to MCP servers, and presents permission surfaces to the user.
whyExists: |
  MCP separates servers (tools/resources) from hosts (user-facing apps). Security and UX live largely in the host.

  Blaming only servers for abuse when the host granted broad auto-approve. Host policy is the control plane.
practicalMeaning: |
  Implement least-privilege defaults, clear consent, server allowlists, and visible tool activity. Treat host security as product security.
