plainDefinition: |
  JSON mode is a vendor feature that steers model output toward syntactically valid JSON objects, usually via decoding constraints or strong instruction bias, without guaranteeing that the JSON matches your application schema.
whyExists: |
  Product teams need machine-readable answers more often than essays. A dedicated mode reduces broken braces and free-text apologies when the next step is a parser.
misuse: |
  Treated as schema validation, security boundary, or proof the model 'understands' structure. JSON mode can still emit wrong keys, wrong types, and confidently empty objects.
practicalMeaning: |
  Use it as a decoding bias, then validate against a real schema, repair or reject failures, and log the miss rate. Prefer true structured-output / schema-constrained decoding when the vendor offers it.

  "JSON mode fixed the curly braces; the invoice total still arrived as a motivational quote."
