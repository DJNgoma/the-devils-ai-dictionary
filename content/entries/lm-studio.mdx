devilDefinition: |
  A desktop studio for running models locally, where downloading the weights is the easy part and fitting them into memory becomes the opening act.
plainDefinition: |
  LM Studio is a desktop and headless runtime for discovering, downloading, running, chatting with, and serving local AI models. It supports GGUF models through llama.cpp, MLX models on Apple Silicon, MCP tools, its own SDKs and APIs, and OpenAI- and Anthropic-compatible endpoints.
whyExists: |
  Local models otherwise require users to assemble runtimes, model files, command-line flags, servers, and interfaces themselves. LM Studio packages those tasks into an approachable application while retaining developer access through a CLI, headless service, and local network APIs.
misuse: |
  LM Studio is confused with a model, a guarantee of privacy, or a fully open-source stack. It is a product that runs separately licensed model weights and runtimes; privacy and openness depend on the model, plugins, network settings, telemetry, and integrations actually used.
practicalMeaning: |
  A user downloads compatible weights, chooses a runtime and memory settings, loads the model, and chats locally or starts a server. Production-minded users must still handle hardware capacity, model licences, authentication, network exposure, updates, evaluation, and the gap between API compatibility and identical behaviour.
