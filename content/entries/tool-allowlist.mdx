plainDefinition: |
  A tool allowlist is the explicit set of tools, operations, and scopes an AI system is permitted to use for a given task, user, environment, or workflow.
whyExists: |
  Agents become safer and easier to reason about when they cannot reach every integration by default. An allowlist turns "the model has tools" into a narrower claim about which tools are available and under which conditions.
misuse: |
  It gets treated as a static checkbox. A generous allowlist with broad write access, production credentials, and no review cadence is not a control. It is an invitation with better formatting.
practicalMeaning: |
  In practice it means named tools, allowed operations, read and write distinctions, environment boundaries, user-specific scopes, approval thresholds, and a review process for removing access once the prototype has stopped pretending to be temporary.
example: |
  "The support agent could read orders and draft refunds, but the tool allowlist kept the actual refund tool behind a separate approval gate."
