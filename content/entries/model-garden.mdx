plainDefinition: |
  Model garden is a vendor term for a managed catalogue of models that customers can browse, provision, fine-tune, and call through a cloud platform.
whyExists: |
  Because buyers want more than one model, clouds want to be the place where that choice gets made, and nobody enjoys integrating a fresh API every time the leaderboard twitches.
misuse: |
  By making model access sound like curation solved the hard part. A catalogue is useful. It does not answer which model is trustworthy, economical, regionally available, or appropriate for a real workload.
practicalMeaning: |
  In practice it usually means a procurement-and-deployment layer with billing, quotas, IAM, regions, logging, and just enough abstraction to turn model shopping into infrastructure policy.
