devilDefinition: |
  The part of the risk register that begins after the model has stopped talking and the connected software starts obeying.
plainDefinition: |
  Tool-layer risk is the risk introduced when an AI system can use external tools, APIs, connectors, or automations that read data, change state, send messages, spend money, or affect other systems.
whyExists: |
  Model output is not always the final hazard. In agentic systems, the serious consequence often appears when a model's structured request reaches ordinary software with real permissions.
misuse: |
  It is misread as another flavour of hallucination risk. A perfectly plausible tool call can still be unsafe if the tool has the wrong scope, stale context, weak validation, or authority the task never deserved.
practicalMeaning: |
  In practice it means mapping every tool to the data it can read, the state it can change, the external parties it can contact, and the controls that stop a bad call from becoming an incident with a timestamp.
example: |
  "The model merely chose the refund tool; tool-layer risk arrived when that tool could issue real payments without checking account policy."
