devilDefinition: |
  A language model that drafts several parts at once, bringing parallelism to the ancient art of revising the sentence after seeing the sentence.
plainDefinition: |
  A diffusion language model generates text by iteratively refining a partially masked or noisy sequence across multiple token positions. Unlike a conventional autoregressive model, it need not commit to every output token strictly from left to right.
whyExists: |
  Iterative parallel refinement can offer different speed, control, and editing trade-offs from sequential decoding. It may allow a model to revise earlier and later parts of an answer together instead of treating every emitted token as irrevocable history.
misuse: |
  The name is borrowed from successful image diffusion systems, which tempts people to assume the same mechanics, quality gains, or economics transfer neatly to language. Text is discrete, ordering matters, and benchmark speed is not the same as useful end-to-end latency.
practicalMeaning: |
  Buyers should inspect output quality at the claimed latency, the number of refinement steps, serving hardware, streaming behaviour, controllability, and performance on tasks where exact order matters. The architectural difference is real; the product advantage must still be measured.
