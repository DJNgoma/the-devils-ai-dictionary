plainDefinition: |
  Web Bot Auth is an approach for letting automated clients prove their identity to websites using HTTP message signatures and associated public-key information. A site can verify that a request was signed by the claimed bot operator instead of relying only on an easily copied user-agent string or source IP.
whyExists: |
  Websites increasingly need to distinguish legitimate search, assistant, commerce, and agent traffic from impersonators and abusive automation. Verifiable requests can support differentiated access, rate limits, policy, and accountability as IP-based identification becomes less reliable.
misuse: |
  Authentication is easily mistaken for approval. A valid signature can show who controls a key; it does not prove that the bot is beneficial, that the user authorised this visit, or that the requested action should be allowed.
practicalMeaning: |
  Bot operators sign defined parts of each HTTP request. Sites retrieve or configure trusted public keys, verify freshness and signature coverage, map the identity to policy, and still apply authorisation, abuse controls, key rotation, and revocation.
