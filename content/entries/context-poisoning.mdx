  Sneaking bad instructions or bad facts into the system's memory and letting future competence be built on polluted paperwork.
plainDefinition: |
  Context poisoning is a failure mode or attack in which retrieved context, shared memory, cached notes, or prior state gets contaminated with false, hostile, or manipulative content that later steers model behaviour.
whyExists: |
  Agent systems increasingly depend on stored traces, retrieved documents, vector indexes, summaries, and shared working memory. Once those layers become operational dependencies, corrupting them becomes a practical way to corrupt the system.
misuse: |
  The term gets overused for any bad answer produced with too much context. The sharper meaning is persistence: poisoned material survives long enough or appears trusted enough to shape future decisions, not just one unlucky response.
practicalMeaning: |
  In practice it means untrusted instructions in documents, planted false facts in memory stores, contaminated summaries, or cross-agent state drift that causes the model to retrieve the wrong thing with total procedural confidence.

Ordinary software forgets many of its mistakes after the request ends. Agent systems increasingly preserve them for later, which is a less charming habit.
