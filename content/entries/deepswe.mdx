devilDefinition: |
  A software-engineering benchmark with enough room for the agent to reveal whether its plan survives contact with an actual repository.
plainDefinition: |
  DeepSWE is DataCurve's benchmark for coding agents on original, long-horizon software-engineering tasks. Its current v1.1 leaderboard covers 113 tasks and reports attempt-level pass@1, task-level pass@4, cost, tokens, duration, steps, and context use across model-and-harness configurations.
whyExists: |
  Patch benchmarks built from familiar public issues can suffer contamination, narrow task scope, and compressed score differences. DeepSWE uses original tasks and longer trajectories to expose differences in repository navigation, implementation, verification, persistence, and harness behaviour.
misuse: |
  DeepSWE is also the name associated with an older DeepSWE-Preview coding agent trained by Agentica and Together AI. Treating that model and DataCurve's newer benchmark as the same object produces citations that evaluate themselves with impressive efficiency.
practicalMeaning: |
  Read the benchmark version, task set, harness, model, reasoning effort, number of attempts, and scoring unit before comparing results. DeepSWE counts context-window failures and agent timeouts as failures while excluding provider, network, and verifier errors from scored attempts.
