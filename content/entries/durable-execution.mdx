devilDefinition: |
  Workflow state written down carefully enough that a crashed process cannot take the institutional memory with it.
plainDefinition: |
  Durable execution is a workflow model that persists progress and can resume long-running work after process crashes, restarts, timeouts, or infrastructure changes. Completed steps and pending waits are recorded outside the transient worker that happens to be running them.
whyExists: |
  Agent jobs increasingly wait on tools, people, schedules, and slow model calls. Keeping that work only in memory makes an ordinary deployment or network failure indistinguishable from forgetting the entire assignment.
misuse: |
  Durable is mistaken for exactly once, automatically correct, or permanently retained. Resumption can repeat side effects, preserve bad state, exhaust retries, or revive work whose user authority has expired.
practicalMeaning: |
  The system checkpoints workflow state, assigns stable operation identifiers, retries according to policy, makes side effects idempotent where possible, and exposes cancellation and inspection. Engineers must version state, bound retention, and decide how old work behaves after code or permissions change.
example: |
  "Durable execution survived the deploy and resumed the agent precisely where it had paused, including the part everyone now wished had been cancelled."
