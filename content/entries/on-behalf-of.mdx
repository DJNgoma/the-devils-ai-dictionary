  Acting for a user with delegated authority—the difference between helpful automation and borrowed identity.
plainDefinition: |
  On-behalf-of describes an agent or service performing actions under delegated authority for a principal user, ideally with scoped tokens and clear audit attribution rather than shared passwords.
whyExists: |
  Agents need to email, file, and purchase for people. Delegation patterns from identity systems became central to agent safety.
misuse: |
  Implemented as 'login as the user' with full privileges. That is impersonation, not delegation.
practicalMeaning: |
  Use OAuth-style delegation, narrow scopes, short lifetimes, user-visible consents, and logs that show agent and principal separately.

  "On-behalf-of access let the agent book travel without holding the CEO password in a prompt."
