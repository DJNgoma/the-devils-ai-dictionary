plainDefinition: |
  An MCP marketplace is a distribution surface where users, developers, vendors, or organisations can discover, publish, install, review, or manage MCP servers and related integrations.
whyExists: |
  MCP turns tool and resource servers into reusable software objects. Once there are enough of them, teams need discovery, packaging, trust signals, versioning, documentation, and a way to decide which integrations deserve to be near a real assistant.
misuse: |
  It gets treated as though marketplace listing equals safety, maintenance, or production readiness. Distribution is not approval. A glossy listing can still expose excessive scopes, stale schemas, weak authentication, or a vendor no one has assessed.
practicalMeaning: |
  In practice it means procurement and security review meeting developer convenience: who authored the server, what it can read or change, how it is updated, what telemetry it emits, who can install it, and how fast it can be removed when enthusiasm ages badly.
