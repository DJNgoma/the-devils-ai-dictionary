devilDefinition: |
  An MCP server moved onto the network, where every convenient tool call acquires latency, credentials, and someone else's uptime.
plainDefinition: |
  A remote MCP server is a network-hosted Model Context Protocol endpoint that exposes tools, resources, or prompts to MCP clients. Unlike a local server process, it sits behind a transport, authentication layer, and operational boundary managed over the network.
whyExists: |
  Local MCP servers are useful for personal tools, but shared services need central deployment, updates, access control, and availability. A remote server lets several approved clients use the same managed integration without installing its implementation on every machine.
misuse: |
  "Remote" is sometimes treated as a deployment detail that changes nothing. It changes the threat model, failure modes, privacy assumptions, latency, tenancy, and the identity to which each tool call is attributed.
practicalMeaning: |
  Teams need to evaluate the server operator, transport security, authorisation scopes, data retention, approval behaviour, rate limits, audit logs, and what happens when the endpoint is slow or unavailable. Standardised discovery does not make a remote capability trustworthy by default.
example: |
  "The remote MCP server saved everyone an installation step and gave security a considerably longer questionnaire."
