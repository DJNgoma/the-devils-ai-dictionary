devilDefinition: |
  A checkpoint placed between intent and consequence, ideally before consequence has already hired a publicist.
plainDefinition: |
  An approval gate is a control point that requires explicit human or system approval before a workflow can perform a sensitive action, proceed to a higher-risk step, or release an output.
whyExists: |
  Agent workflows can read data, call tools, spend money, send messages, change records, or trigger downstream systems. Approval gates exist because some actions deserve a pause, evidence, and a responsible decision before automation turns possibility into fact.
misuse: |
  It gets treated as safety by interface design. A button labelled "approve" is not a meaningful gate if the reviewer lacks context, cannot reject without penalty, or is asked after the important action has already happened.
practicalMeaning: |
  In practice it usually means risk thresholds, reviewer context, audit records, timeout behaviour, rejection paths, escalation rules, and a clear distinction between approval as control and approval as decoration.
example: |
  "The approval gate stopped the agent from sending the refund email, though it could do nothing about the three-paragraph apology it had already composed for morale."
