  The serving trick where you pay full price once so the model can stop rereading your sacred system prompt on every request.
plainDefinition: |
  Prompt caching is a technique where a provider reuses the already processed prefix of repeated prompts, reducing latency and input cost when the beginning of the request stays the same.
whyExists: |
  It exists because long instructions, tool schemas, examples, and reference context are expensive to process from scratch every time. If large parts of a prompt repeat, caching lets systems avoid doing the same preparatory work over and over.
misuse: |
  It gets pitched as free speed or magical memory. In reality it only helps when prompts share the right prefix, the provider's cache keeps it warm, and the traffic pattern is disciplined enough to hit it again before it expires.

  In practice it usually means putting stable instructions first, volatile user data later, watching cache hit rates, and discovering that prompt order has quietly become a finance decision.
