  An MCP registry is a catalogue or discovery surface for MCP servers, tools, resources, prompts, or integrations that clients and organisations can find, evaluate, and configure.
whyExists: |
  As MCP adoption grows, teams need a way to discover available servers and understand what each one exposes. A registry can reduce integration guesswork and make capability discovery less tribal.
misuse: |
  It gets treated like a marketplace where listing implies trust. Discovery is not approval, security review, maintenance, or suitability for production use.
practicalMeaning: |
  In practice it means procurement, security review, version tracking, owner metadata, permission policies, and a process for removing servers that should no longer be near an assistant.
