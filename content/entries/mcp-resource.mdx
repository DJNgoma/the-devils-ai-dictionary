plainDefinition: |
  An MCP resource is application-provided context exposed through the Model Context Protocol, such as a file, document, record, page, or other readable item that a client can discover or request.
whyExists: |
  MCP separates resources from tools. Tools do things; resources provide context. That distinction helps clients browse, retrieve, display, and reason about external information without turning every read into an action.
misuse: |
  It gets blurred into tool use, as though every available resource is something the model may freely consume. Resources still need permissions, scoping, metadata, and a reason to be used.
practicalMeaning: |
  In practice it means a server can expose relevant context in a standardised way, while the client decides what to show, fetch, or pass into the model for a specific task.

  "The MCP resource exposed the project brief to the assistant; it did not authorise the assistant to rummage through payroll because curiosity felt agentic."
