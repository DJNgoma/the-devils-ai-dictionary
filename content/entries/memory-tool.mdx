devilDefinition: |
  A file cabinet the model may update between conversations, with filing standards supplied by whichever application volunteered to become the archivist.
plainDefinition: |
  A memory tool is a client-implemented tool that lets a model read and modify persistent information outside the current conversation. In Anthropic's named implementation, the client provides file-style operations and storage while the model decides what memory to create, retrieve, update, or delete.
whyExists: |
  A context window ends with the request, while useful assistants need preferences, project state, and learned instructions to survive across sessions. A tool gives the model explicit operations over durable storage instead of pretending all prior conversation can remain in every prompt.
misuse: |
  Memory is marketed as though persistence implies accurate recall. A model can store the wrong conclusion, retrieve stale instructions, expose sensitive material, or keep a confident summary long after the underlying facts have changed.
practicalMeaning: |
  The application owns the storage boundary, access controls, retention, deletion, and synchronisation. It should show users what is remembered, separate scopes, defend memory writes from prompt injection, and verify important state against current sources before acting.
example: |
  "The memory tool remembered the user's preferred report format and the obsolete project deadline with equal devotion."
