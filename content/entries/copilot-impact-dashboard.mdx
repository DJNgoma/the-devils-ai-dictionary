  A dashboard that turns AI adoption into a maturity journey and correlation into executive-ready scenery.
plainDefinition: |
  GitHub's Copilot impact dashboard is an enterprise view that groups engaged users by Copilot adoption phase and displays measures such as merged pull requests, merge velocity, lines of code, cohort size, and trends relative to licensed but unengaged users.
whyExists: |
  Licence counts and monthly-active-user totals reveal little about how developers use coding assistance. Cohorts can show whether use remains code-completion-heavy, moves towards agents, or expands into multi-agent and Copilot-app workflows.
misuse: |
  Differences between passive and engaged cohorts are presented as productivity caused by Copilot. Self-selection, team mix, repository difficulty, seniority, process changes, code quality, review burden, incidents, and work displaced outside GitHub can all produce the same attractive chart.
practicalMeaning: |
  Use the dashboard to inspect adoption and form hypotheses, not to calculate return on investment alone. Pair it with controlled comparisons, quality and reliability measures, developer experience, security findings, review time, rework, and business outcomes.
