plainDefinition: |
  An inference budget is the planned or enforced limit on the cost, latency, tokens, model calls, compute, or provider spend used to run AI features in production.
whyExists: |
  Model calls have variable cost and performance characteristics. Without a budget, a successful feature can become a financial incident, and an agent can discover new forms of curiosity at the company's expense.
misuse: |
  It gets reduced to token counting. Tokens matter, but so do model choice, retries, tool calls, reasoning effort, cache hits, latency targets, and the number of times a workflow politely tries again.
practicalMeaning: |
  In practice it means per-user limits, per-task ceilings, model routing policies, cache strategy, timeout rules, alerting, and deciding which requests deserve the expensive model before the expensive model decides for itself.

  "The inference budget survived launch only after the agent was stopped from spending premium reasoning on password reset emails."
