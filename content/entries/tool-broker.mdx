devilDefinition: |
  The middleman deciding which external powers the agent may see, request, or be politely denied before curiosity becomes consequences.
plainDefinition: |
  A tool broker is the component or service that mediates which tools an agent can discover, select, or invoke, often based on permissions, context, policy, or environment.
whyExists: |
  Agents rarely need unrestricted visibility of every available capability. A broker exists to separate tool exposure from tool implementation, making it easier to apply permissions, environment boundaries, capability filtering, or contextual access rules.
misuse: |
  It gets described as though brokering itself guarantees security or good judgement. It does not. A broker can make tool access more governable, but it can also become a fashionable wrapper around the same old permission mistakes if the policy behind it is sloppy.
practicalMeaning: |
  In practice it usually means registries, capability filters, tenancy rules, environment-aware allowlists, protocol adapters, and a decision point that determines whether the agent gets a calculator, a search index, a payment action, or a polite refusal.
example: |
  "The tool broker was the quiet adult in the room. It let the agent discover the search tool and blocked the urge to improvise with production billing."
