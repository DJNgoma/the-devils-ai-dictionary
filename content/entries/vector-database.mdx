  A database marketed with unusual enthusiasm because "we store numerical similarity indexes" never raised venture money on its own.
plainDefinition: |
  A vector database is a system designed to store and query vector embeddings efficiently, usually to support similarity search and retrieval over large collections of items.
whyExists: |
  Standard databases can store vectors, but specialised systems often provide faster approximate nearest-neighbour search, filtering, indexing, and operational features tuned for embedding-driven retrieval workloads.
misuse: |
  It gets talked about as though adding a vector database automatically gives a product memory, understanding, or RAG competence. It does not. It gives you a place to store and search embeddings. The quality of the overall system still depends on chunking, retrieval logic, reranking, and task design.
practicalMeaning: |
  In practice it usually means one component in a retrieval stack: embeddings go in, candidate matches come out, and the rest of the application still has to decide whether those matches are relevant enough to trust.
