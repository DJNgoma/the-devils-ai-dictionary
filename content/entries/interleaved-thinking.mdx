devilDefinition: |
  Reasoning allowed to continue after the tool replies, a modest concession to the possibility that reality has amended the original plan.
plainDefinition: |
  Interleaved thinking is a model execution pattern in which reasoning occurs between tool calls as well as before the first call. The model can inspect each result, revise its approach, and decide what to do next instead of following a fully formed tool plan unchanged.
whyExists: |
  Tool results reveal errors, missing data, changed state, and better routes that cannot be known in advance. Reasoning between calls lets an agent adapt to evidence during a task rather than using tools merely to decorate its initial guess.
misuse: |
  It is treated as a guarantee of better judgement or autonomous reliability. More reasoning opportunities can also mean more tokens, latency, wandering plans, unjustified tool calls, and a longer record of the model adapting badly.
practicalMeaning: |
  The runtime preserves the model's reasoning and tool context across a sequence, returns each tool result in the required form, and enforces budgets, approvals, and stop conditions around the loop. Teams should evaluate task success and side effects, not the apparent sophistication of the trace.
